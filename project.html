The brief:


I am reviewing some manuals...they have some mistakes..

I need to deploy an app that lets a team make a record for each mistake in a document. Each record would need a document reference (a paragraph number e.g. 1.2.34.a)/ the original text(people would cut and paste large data) / an error type / a comment / a revision suggestion (could also be a large cut and paste)/ created by / and timestamp, these are all string fields.

The important bit: each entry needs an approved status and an incorporated column which would be boolenas.

It also needs devise for user auth. A single user class is probably enough

Aside for CRUD capability for each entry, the only real magic is that I need a view that displays all the entries that have been aproved for change but not yet incorporated. we will send this to the document amendment team to change the manuals. I need the same database for 3 manuals

Timing for a MVP is ASAP as usual
